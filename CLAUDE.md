# logic-frameworks プロジェクト指示書

## プロジェクト概要
ビジネスフレームワーク（マトリクス図、ロジックツリー、ベン図など）を実際に編集・保存できるWebツール集。
最終的に85種類を収録し、一般公開する。

## 技術スタック
- HTML / CSS / バニラJavaScript（外部フレームワーク不使用）
- データはdata/frameworks.jsonで一元管理
- 保存はブラウザのlocalStorageを使用（v1はローカル保存のみ）

## 実装方針（重要）
85種類のフレームワークは、見た目の「型」によって以下の6カテゴリに分類して実装する。
個別に85個作るのではなく、型ごとに1つの汎用エンジンをjs/engines/に作り、
frameworks.jsonのデータを流し込む形で展開する。

- 型A：マトリクス型（軸×軸の4象限など）→ matrix.js
- 型B：ツリー/階層型（ロジックツリーなど）→ tree.js
- 型C：プロセス/フロー型（PDCAなど）→ flow.js
- 型D：記入式シート型（5W1Hなど）→ sheet.js
- 型E：円/集合図型（ベン図など）→ venn.js
- 型F：時系列/線表型（ガントチャートなど）→ timeline.js

## 著作権ルール（厳守）
各フレームワークの解説文・サンプル例文は、他サイトや書籍の文章を一切参照・転載せず、
フレームワークの一般的な構造・定義のみを参考に完全オリジナルで作成すること。

## デザインルール
- メインカラー：#2563EB（知的で信頼感のある青）
- フォント：Noto Sans JP
- 編集可能な要素は枠線をはっきりさせ、「ここが編集できる」と直感的に分かるUIにする

## Git運用ルール
- 1つの型エンジンが完成するごとにコミットすること
- コミットメッセージは日本語で内容を明記すること

## エンジン拡張のスコープ方針（ステップ3で確定）

フレームワークは「6つの汎用エンジン（matrix/tree/flow/sheet/venn/timeline）＋ sheet の
grid2x2 レイアウト」で賄うことを基本とし、型の経済性を優先する。1フレームワークのために
専用エンジンを乱立させない。

- 新規エンジンは原則追加しない（型は上記6＋grid2x2で打ち止め）。
- 既存型に素直に乗らない特殊ケースは、無理に押し込まず、以下の方針で扱う：
  - フィッシュボーン（特性要因図）：将来拡張として保留。需要が確実になれば専用エンジンを
    単独タスクで個別検討する（斜め骨の描画＝SVG座標計算が難所のため慎重に）。
  - 分岐フロー／フローチャート（条件分岐）：収録対象外。本ツールは「思考フレームワーク」が
    主旨で、業務手順図は範囲外。flow の分岐拡張は費用対効果が低い。
  - その他（散布図・ファネル・比較表など）：データ投入時に都度判断。既存型で近似できるものは
    それで扱う（例：ファネルは flow の linear で近似の見込み）。
- 結論：85種類の完全網羅より、型に素直に乗るものを優先して品質を保つ。

### データ投入の採用基準（ステップ5）
収録数の目安（参考ページ上の約85種）はクォータではない。各候補は投入前に次の4条件を満たすこと。満たさないものは件数のために採用せず見送る。
1. 既存6型＋ grid2x2 に素直に乗る（無理に押し込まない）。
2. 名前と一般構造のみ参照し、解説・例文は完全オリジナル（著作権ルール準拠）。
3. 思考ツールとして実質が成立する（枠だけの水増しにしない）。
4. 既存フレームワークや他候補と機能的に重複しない。
各カテゴリは良質な候補が尽きたらそこで打ち止め。少なすぎは不可だが、85に合わせるための質の低い収録は避ける。
